¿Qué es un DNS?

Domain Name System · Sistema de Nombres de Dominio

Es la "agenda telefónica" de internet: traduce los nombres que escribimos en direcciones IP que las máquinas entienden.

El problema que resuelve

Tú recuerdas fácil:

www.google.com

➜ DNS ➜

La máquina necesita:

142.250.190.78

Sin DNS tendrías que memorizar números para cada sitio web.

¿Cómo funciona? (paso a paso)

1

Escribes una dirección en el navegador.

2

Tu equipo pregunta al servidor DNS por la IP.

3

El DNS responde con la IP correspondiente.

4

El navegador se conecta a esa IP y carga la página. ¡En milisegundos!

La jerarquía del DNS

1. Servidores raíz (root)

El punto de partida de toda consulta.

↓

2. Servidores TLD

Gestionan las terminaciones: .com, .org, .ec …

↓

3. Servidor autoritativo

Tiene la IP final del dominio buscado.

Tipos de registros DNS

A

Nombre → dirección IPv4.

AAAA

Nombre → dirección IPv6.

CNAME

Alias de un nombre a otro.

MX

A qué servidor llega el correo.

TXT

Texto libre: verificaciones y seguridad de correo (SPF, DKIM).

Caché DNS y TTL

Para no preguntar siempre lo mismo, las respuestas se guardan temporalmente (caché). El TTL dice cuánto tiempo guardarlas.

💾

Más rápido

Webs visitadas cargan al instante.

⏲️

TTL

Tiempo de vida del dato guardado.

🧹

Limpiar caché

ipconfig /flushdns

Problemas comunes

⚠️ "DNS server not responding"

El servidor DNS falla; prueba cambiar a 8.8.8.8 o 1.1.1.1.

🐢 La web no carga pero el internet "anda"

Suele ser caché vieja: usa ipconfig /flushdns.

🌍 Una web se ve en un país y en otro no

Propagación de DNS: los cambios tardan en actualizarse en todo el mundo.

Seguridad del DNS (2026)

🔏 DNSSEC

Firma las respuestas para evitar que te engañen con una IP falsa.

🔒 DNS over HTTPS (DoH)

Cifra tus consultas para que nadie vea qué sitios visitas.

🎣 DNS spoofing / phishing

Ataques que te redirigen a webs falsas. DNSSEC ayuda a evitarlos.

🛡️ DNS con filtro

Algunos bloquean malware y contenido inseguro automáticamente.

Resumen del tema

El DNS traduce nombres en direcciones IP.
Funciona en jerarquía: root → TLD → autoritativo.
Usa registros (A, AAAA, CNAME, MX, TXT) y caché/TTL.
Se protege con DNSSEC y DNS over HTTPS.
